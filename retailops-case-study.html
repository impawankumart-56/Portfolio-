Enterprise Operations

Overview

This project focused on redesigning a multi-module enterprise SaaS platform used for:

  • field operations,
  • approvals,
  • asset management,
  • collections,
  • operational tracking,
  • and administrative workflows.

The product was already live across multiple enterprise clients. The challenge was improving consistency and usability without affecting existing workflows.

The Problem

The platform had expanded over time through multiple operational modules and client requirements.

This resulted in:

  • inconsistent layouts,
  • repeated interaction patterns,
  • varying UI behaviors,
  • and higher cognitive effort for users moving between workflows.

Operations teams were using the product daily in high-frequency environments where speed and clarity were critical.

Scope

The redesign included:

  • ~100 production screens,
  • operational workflows,
  • approval systems,
  • asset lifecycle flows,
  • and responsive web experiences.
Design Approach
Design System Foundation
Used Microsoft Fluent UI to establish: consistent layouts, reusable components, scalable spacing systems, and predictable interaction behavior. This created a stronger design foundation for future scaling.
Workflow-Focused UX
The redesign focused on: reducing navigation effort, improving workflow continuity, simplifying repetitive tasks, and improving operational visibility. The goal was to make the product feel more connected across modules.
Simplified Enterprise Workflows
Improved: approvals, asset requests, lifecycle tracking, collections, and operational visibility through clearer layouts and more structured interactions.
UX Principles Applied
Design System Thinking Workflow Continuity Operational Clarity Reduce Cognitive Load Scalable UX
Outcome
  • ~100 screens redesigned
  • Improved consistency across modules
  • Better workflow visibility
  • More scalable design structure established

The redesign is currently under review as part of a larger enterprise rollout.

Reflection

This project became a strong learning experience in designing production-scale SaaS systems.

It reinforced the importance of:

  • scalable design thinking,
  • system consistency,
  • and designing for long-term operational growth rather than isolated screens.
